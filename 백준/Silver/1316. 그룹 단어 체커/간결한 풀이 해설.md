동작 원리를 단계별로 뜯어보겠습니다.

---

### 1. 코드 분석

```python
text = "banana"
result = sorted(text, key=text.find)

```

* **대상(`text`)**: `"banana"`라는 문자열을 한 글자씩 분리한 리스트 `['b', 'a', 'n', 'a', 'n', 'a']`를 정렬 대상으로 삼습니다.
* **기준(`key=text.find`)**: 각 글자를 정렬할 때, 그 글자가 원본 문자열(`text`)에서 **처음으로 나타나는 인덱스**를 기준으로 삼습니다.

### 2. 단계별 실행 과정 (예시: "banana")

`sorted` 함수가 각 문자를 검사할 때 `text.find(문자)`를 호출하여 그 결과값(인덱스)을 비교합니다.

| 문자 | `text.find(문자)` 계산 (정렬 기준값) | 설명 |
| --- | --- | --- |
| 'b' | `text.find('b')` → **0** | 'b'는 인덱스 0에서 처음 등장 |
| 'a' | `text.find('a')` → **1** | 'a'는 인덱스 1에서 처음 등장 |
| 'n' | `text.find('n')` → **2** | 'n'는 인덱스 2에서 처음 등장 |
| 'a' | `text.find('a')` → **1** | 여전히 처음 등장 위치인 1을 반환 |
| 'n' | `text.find('n')` → **2** | 여전히 처음 등장 위치인 2을 반환 |
| 'a' | `text.find('a')` → **1** | 여전히 처음 등장 위치인 1을 반환 |

### 3. 최종 정렬 결과

정렬 기준값(인덱스) 순서대로 문자를 나열하면 다음과 같습니다:

1. 기준값 **0**인 문자: `'b'`
2. 기준값 **1**인 문자들: `'a'`, `'a'`, `'a'`
3. 기준값 **2**인 문자들: `'n'`, `'n'`

따라서 `result`는 `['b', 'a', 'a', 'a', 'n', 'n']`이 됩니다.

---

### 4. 이 코드의 특징: "중복 문자 모으기"

이 코드는 알파벳 순서가 아니라, **원본에서 먼저 나온 문자 순서대로 끼리끼리 뭉치게 만드는 효과**를 줍니다.

* `"banana"` → `['b', 'a', 'a', 'a', 'n', 'n']`
* `"apple"` → `['a', 'p', 'p', 'l', 'e']` (p가 1번 인덱스에서 처음 나왔으므로 p끼리 뭉침)
* `"mississippi"` → `['m', 'i', 'i', 'i', 'i', 's', 's', 's', 's', 'p', 'p']`

> **주의할 점:**
> `sorted()`는 리스트를 반환하므로, 다시 문자열로 만들고 싶다면 `"".join(sorted(text, key=text.find))`와 같이 사용해야 합니다.

이 방식은 데이터의 **"출현 순서 유지"**가 중요할 때 유용하게 쓰일 수 있습니다.
